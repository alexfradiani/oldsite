PintArte
App de dibujo
Facebook
(2010)

Funciones desempeñadas:

- Programación

- Participación en la creación de concepto
Descripción
PintArte es una utilidad de dibujo desarrollada como aplicación Facebook, a la cual cualquier usuario puede acceder desde la red social y realizar sus propios artes.

El software utiliza una imagen que puede ser cargada por el usuario desde sus imagenes locales, desde sus álbumes de fotos en facebook, o desde la cámara web si la tiene activada. Una vez cargada la imagen puede dibujar sobre ella utilizando herramientas de pincel de colores, y colocar accesorios.

En este sistema se utilizaron las tecnologias flash AS3, php, Mysql para el desarrollo de la interfaz visual, comunicacion con servidor de base de datos y conexión a la red social.

CicloEconómico
Juego Educativo
(2010)

Funciones desempeñadas:

- Programación

- Participación en la creación de concepto
Descripción
"Ciclo Económico" es un juego educativo. Consta de dos subjuegos cada uno con tres escenas. Las escenas se cargan por medio de parámetros XML con coordenadas y tamaños de los sprites de los personajes.

En cada escena el jugador puede moverse dentro de un espacio 2D pero que simula una perspectiva en 3D (al estilo "monkey island") haciendo click. Se pueden establecer diálogos con los personajes presentes en la escena.

Al mismo tiempo el jugador cuenta con un inventario de "items" que puede intercambiar con esos personajes. Tanto los items como los diálogos tienen unas reglas de lógica de acuerdo a lo que se debe entregar para recibir cierto item a cambio y seguir avanzando.

Museo
Galería virtual
(2010)

Funciones desempeñadas:

- Programación

- Participación en la creación de concepto
Descripción
Esta es una galería virtual de los elementos de un museo. Los objetos están inicialmente flotando sobre espacio vacío. Al hacer click hay un tween de zoom hacia el objeto y se cargan otros elementos que pertenecen a esa colección. cuando se carga el objeto viene un menú en forma de "carrusel" donde se puede escoger el multimedia disponible sobre ese objeto.

Esta información también es dinámica, las imágenes y el contenido video/audio se carga de forma externa y está descrito en un archivo XML. Cambiando los valores del archivo se puede ajustar el contenido.

Wartsila
Software informativo
Juego ilustrativo
(2010)

Funciones desempeñadas:

- Programación

- Participación en la creación de concepto
Descripción
Este sistema tiene el propósito de presentar información en forma interactiva sobre las actividades de una empresa energética, para ello se desarrollaron tres módulos diferentes.

El primero servía como juego de construccion de plantas, donde se muestra un plano isométrico sobre el cual se pueden colocar diferentes tipos de construcciones que se encuentran en una librería de items; El usuario dispone de ella para crear su propio arreglo industrial, a medida que se colocan los elementos se ajustan valores enérgeticos estadísticos asociados.

Los dos módulos restantes aportaban una galería visual, en la que se presentan una serie de imágenes cargadas externamente, y un mapa interactivo en el que se pueden apreciar las plantas y cargar desde allí configuraciones precargadas del módulo de construcción.

Quiniela
Sudáfrica
(2010)

Funciones desempeñadas:

- Programación
Descripción
Esta es una aplicación Web creada para hacer seguimiento de los resultados de la copa mundial de Fútbol Sudáfrica 2010. Se presentan dos módulos con los cuales se puede observar la evolución en resultados de todos los juegos que conformaron la competencia.

Se cuenta con un simulador incorporado dentro de la aplicación, con el cual se podría anticipar el arreglo de equipos en los juegos a lo largo de todas las rondas, con solo introducir los resultados de los primeros juegos. Esto se hizo incorporando las reglas reales por las que el campeonato se rige en cada una de sus fases (1era ronda, octavos de final, etc...) y cargando en una base de datos la secuencia respectiva de encuentros que fue preparada por la FIFA para este mundial.

Simulación
(2010)

Funciones desempeñadas:

- Programación

-Adaptación de tecnologías.
Descripción
Este Sistema es un prototipo creado para simular las dinámicas de juego "Penalty".

En este proyecto se utilizó la tecnología de sensores infrarojos original de la consola Wii, esto se hizo adaptando las librerías de decodificación del control Wiimote desarrolladas por Jhonny lee, en combinación con la plataforma Flash AS3 y la librería de dibujo 3d Papervision.

- Participación en creación de concepto
Descripción
El Sistema ofrece un directorio comercial, en el que se pueden consultar los diferentes locales comerciales y su información dentro de un centro comercial.

Se dispone de un conjunto de planos en los que se representa las áreas que conforman la construcción; En estos planos se pueden observar en perspectiva y rotar sobre un modelo 3D todos los locales del centro comercial. PSO
Software
informativo
(2009)

Funciones desempeñadas:

- Programación

- Participación en creación de concepto
Descripción
El proyecto PSO es un Software informativo en el que se puede navegar a través de un mapa interactivo y obtener datos de proyectos industriales, agrícolas, urbanos, etc en la faja petrolífera del Orinoco.

El menú principal de la aplicación permite seleccionar el tipo de proyectos, los cuales tienen un grupo de íconos asociados en el mapa, al escoger algún subgrupo se ajusta dinámicamente los íconos que se muestran sobre el territorio para filtrar la categoría seleccionada.

Por medio del mapa se puede acceder a información multimedia, relacionada a la zona y al tipo de proyectos en desarrollo dentro de ese punto. Adicionalmente se puede filtrar por 4 polos (Ayacucho, Junín, Boyacá, Carabobo) el área de navegación y consultar los diferentes proyectos en ese sector.

ProliSanta
AdverGame
(2009)

Funciones desempeñadas:

- Programación

- Participación en la creación de concepto
Descripción
Este es un "Advergame" utilizado como herramienta de mercadeo para ofrecer a los usuarios un premio adicional como resultados de compras en un punto de venta.

El juego utiliza una simulación simple de física en el objeto de regalo cayendo y la colisión con la nube, y luego cuando llega a la chimena es succionado. Se empleo la técnica de "timers" y "render" cada ciertas fracciones de segundos, en donde se aplicaban los cambios de coordenadas del regalo, las nubes y los otros elementos.

También se emplea control de datos de los premios que vienen de una base de datos Mysql, ya que debían estar limitados a un inventario. se utilizó PHP en el lado servidor para estas consultas.

Domino
AdverGame
(2008)

Funciones desempeñadas:

- Programación

- Participación en la creación de concepto
Descripción
El juego de dominó es un "Advergame" con caracteristicas similares al proyecto anterior "Prolisanta" (tiempo de juego reducido por parte del usuario, premio al finalizar la jugada, base de datos relacionada con existencia de premios en inventario).

El modo de juego simula al clásico juego de mesa de dominó, pero simplificado con reglas de lógica programadas para que el usuario siempre ganara y no se bloqueara una partida.

El usuario se enfrenta al computador, al iniciar escoge tres piezas, los pares de números que originalmente tiene una pieza son reemplazados por imágenes que representan los posibles premios, y que son cargadas en base a la data presente en el inventario. Al final de cada jugada el usuario puede decidir si se lleva el premio identificado con su pieza o continua jugando.

Fútbol
AdverGame
(2009)

Funciones desempeñadas:

- Propuesta de concepto

- Programación
Descripción
Este "Advergame" tiene el propósito de servir como herramienta de mercadeo, la interacción del juego es muy sencilla solicitando del usuario que dispare el balón y espere a recibir un premio.

La programación de la aplicación consiste de un módulo de simulación de física de la pelota, al rebotar sobre las paredes y colisionar con las piezas de otros jugadores.

Existen dos posibles resultados, la pelota choca contra alguno de los arqueros, por lo que el usuario pierde y debe intentar de nuevo, o entra en alguno de los arcos que tienen imágenes de premios, en este caso se dispara una animación identificando el premio recibido. Se utilizaron tecnologías de servidor PHP, Mysql para datos de inventario, y AS3 para la programación del juego.
